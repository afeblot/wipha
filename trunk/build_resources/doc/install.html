INSTALLING

To install WiPhA, just run the WiPhA installer application. You will be prompted for an administrator login/password, since the installer will modify the web server configuration. At the end of the installation, the web server will be started or restarted and the WiPhA login page will open in your browser (bookmark the address for future easy use).

WiPhA is not a mac application. It's a web site, and thus, is installed as a new subdirectory of your personal website directory: /Users/<user>/Sites/wipha

<user> is your short user name. See below for more information.

For those who don't like to give their admin password without knowing what will be done, here is the exact list of actions requiring these privileges:

  • Lets Apache deal with php (not default enabled by Apple)
    • uncomments 2 lines in /etc/httpd/httpd.conf and backups your previous file
  • Enables local .htaccess file to protect WiPhA from external attacks without modifying the general configuration
    • adds 3 lines at the end of /etc/httpd/httpd.conf and backups your previous file if not already done

Other permission modifications performed by the installer:

  • Sets WiPhA data directories so that Apache (www) can write in them (user logins, thumb cache)
    • chmod 3777 data data/cache (777 + setgid + sticky)
  • Sets the setuid bit (with user right, not root!) on a small C program which will chmod o+r or o+x the required iPhoto Library files or directories. This has proved bo be needed for different iPhoto versions which remove these rights on every modification or file creation, whatever your umask is (I hate that).
    For security reason, this program can only be run by you and Apache (www).
    • chmod 4755 changeperm
Should you move this C program which corrects permissions, would it loose it's priviledge to perform it's task. It would then be likely that new modified photos wouldn't be accessible to WiPhA. WiPhA ensures that the proper permissions are set and will warn you (and stop) if they're not. This issue can easily be solved:
  • Open the Terminal program (in /Applications/Utilities)
  • type the following command: chmod 4755 /Users/<user>/Sites/wipha/changeperm
You can also simply re-install WiPhA which will set this again.

FOR THE RECORD

The installer does start the Web server and launches WiPhA in your browser. Would you want however to perform these (simple) operations by yourself, here's how to do it.

STARTING

You will use 3 System Preference panels.

In the system preferences panel, choose the Share panel 3.

Share panel 3: click on the personal web share item to start the server.

CONNECTING TO WIPHA

Now, you just have to know which URL to type in your browser to discover WiPhA. This URL has the form:

http://your_IP_address/~your_short_user_name/wipha

you'll get the ~ character by typing Option+n then space
When the server is started, the Share panel 3 may display an URL to access your web site. This URL uses a host name equivalent to your IP address.

Anyway, if this is not displayed, you may find your IP address in the system preference Network panel 1, and your short user name in the Accounts panel 2.

Network panel 1: find your IP address

Accounts panel 2: find your short user name

So you have now all you need to know your URL to WiPhA. Get Safari, or your preferred browser and go to this address to obtain the WiPhA login page.
